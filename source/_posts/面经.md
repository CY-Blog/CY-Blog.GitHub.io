## 驱动中操作物理绝对地址为什么要先ioremap?
示例代码：
``` c
	IMX6U_CCM_CCGR1 = ioremap(regdata[0], regdata[1]);
	SW_MUX_GPIO1_IO03 = ioremap(regdata[2], regdata[3]);
  	SW_PAD_GPIO1_IO03 = ioremap(regdata[4], regdata[5]);
	GPIO1_DR = ioremap(regdata[6], regdata[7]);
	GPIO1_GDIR = ioremap(regdata[8], regdata[9]);
```
因为linux内核操作的都是虚拟地址 所以要进行一部映射才能访问到对应地址 但是在后买你的基于设备树的驱动上 pinctrl已经为我们完成了电器属性和引脚复用 gpio已经为我们完成了设备的具体引脚 做好了这些事情 我们不需要自己在驱动文件里进行影射了 我们只需要在对应的设备里使用对应的api`gpio_set_value` 类似这种

## copy_to_user()和copy_from_user()主要用于实现什么功能?一般用于file_operations结构的哪些函数里面?
因为：我们的内核和用户数据是不能互相访问的 所以要访问必须经过内核的api函数copy_to_user():完成内核空间到用户空间的复制，copy_from_user()：是完成用户空间到内核空间的复制。一般用于file_operations结构里的read,write,ioctl等内存数据交换作用的函数。当然，如果ioctl没有用到内存数据复制，那么就不会用到这两个函数。

主设备号是定义了驱动的id 比方说我可以注册一个驱动的驱动id是200 那么从设备就是一个具体的led等 比如我 `mknod /dev/led0  200 1 ` 就是把led0 这个设备 联系上了200 这个驱动 然后进行操作 我可以还注册 `mknod /dev/led1  200 2 `
## 设备驱动程序中如何注册一个字符设备?分别解释一下它的几个参数的含义
就直接说现在用的吧
先创建设备号
```c
 if (newchrled.major) { /* 定义了设备号 */
 newchrled.devid = MKDEV(newchrled.major, 0);
 register_chrdev_region(newchrled.devid, NEWCHRLED_CNT, 
NEWCHRLED_NAME);
 } else { /* 没有定义设备号 */
 alloc_chrdev_region(&newchrled.devid, 0, NEWCHRLED_CNT,
NEWCHRLED_NAME); /* 申请设备号 */
 newchrled.major = MAJOR(newchrled.devid); /* 获取主设备号 */
 newchrled.minor = MINOR(newchrled.devid); /* 获取次设备号 */
 }
```
创建新的dev注册
```c
 newchrled.cdev.owner = THIS_MODULE;
 cdev_init(&newchrled.cdev, &newchrled_fops);

 /* 3、添加一个 cdev */
 cdev_add(&newchrled.cdev, newchrled.devid, NEWCHRLED_CNT);
```
创建类创建设备
```c

 newchrled.class = class_create(THIS_MODULE, NEWCHRLED_NAME);
 if (IS_ERR(newchrled.class)) {
 return PTR_ERR(newchrled.class);
 }
 
 newchrled.device = device_create(newchrled.class, NULL,
newchrled.devid, NULL, NEWCHRLED_NAME);
 if (IS_ERR(newchrled.device)) {
 return PTR_ERR(newchrled.device);
 }
```
