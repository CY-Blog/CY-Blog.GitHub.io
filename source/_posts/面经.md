5. 数组和链表区别
    - 存储方式：数组是一段连续的内存空间，用于存储相同类型的元素；链表是由节点组成，每个节点包含数据和指向下一个节点的指针。
    内存分配：数组在创建时需要分配固定大小的内存空间，而链表可以动态分配节点，根据需要进行扩展或缩小。
    插入和删除操作：数组在中间插入或删除元素时需要移动其他元素，而链表只需要修改指针即可。
    访问效率：数组通过下标可以直接访问元素，访问效率较高；链表需要从头节点开始遍历，访问效率较低。
    内存占用：数组的大小固定，可能造成内存浪费；链表根据实际需要动态分配内存，更加灵活。

4. static和const的区别，常用在什么地方，空间上有什么区别吗
    - static关键字用于修改变量和函数的作用域。在全局变量中，static关键字使变量的作用域限定在当前文件内，即变量对其他文件是不可见的。在局部变量中，static关键字使变量的生命周期延长到整个程序执行期间，而不是只在其定义的块中有效。
    const关键字用于声明常量，即数值在程序执行期间不可修改。常量的值在初始化后不能再被改变，这样可以增加代码的可读性和安全性。const关键字通常用于修饰变量、函数参数和函数返回值，以确保它们不被意外修改。

>16. 如何实现让进程A先执行，进程B再执行
    - 锁机制 
    ```c
    #include <stdio.h>
#include <pthread.h>

pthread_mutex_t mutex;

void* processA(void* arg) {
    // 进程 A 执行的任务
    printf("Process A\n");

    pthread_exit(NULL);
}

void* processB(void* arg) {
    // 等待获取锁
    pthread_mutex_lock(&mutex);

    // 进程 B 执行的任务
    printf("Process B\n");

    // 释放锁，允许其他进程获取
    pthread_mutex_unlock(&mutex);

    pthread_exit(NULL);
}

int main() {
    pthread_t threadA, threadB;

    // 初始化锁
    pthread_mutex_init(&mutex, NULL);

    // 创建进程 A
    pthread_create(&threadA, NULL, processA, NULL);

    // 等待进程 A 完成任务
    pthread_join(threadA, NULL);

    // 创建进程 B
    pthread_create(&threadB, NULL, processB, NULL);

    // 等待进程 B 完成任务
    pthread_join(threadB, NULL);

    // 销毁锁
    pthread_mutex_destroy(&mutex);

    return 0;
}
    ```

16. OSI4层模型？分别列举每层都有哪些协议？
     - 

    第一层：物理层（Physical Layer）
        负责在物理媒介上传输原始比特流。
        协议示例：Ethernet、RS-232、USB、Wi-Fi等。

    第二层：数据链路层（Data Link Layer）
        提供可靠的数据传输，将原始比特流转化为帧，并进行错误检测和纠正。
        协议示例：Ethernet（IEEE 802.3）、Wi-Fi（IEEE 802.11）、PPP、HDLC等。

    第三层：网络层（Network Layer）
        负责在网络中选择和控制数据的路径，并进行逻辑地址分配和路由。
        协议示例：IP（Internet Protocol）、ICMP、ARP、RIP、OSPF等。

    第四层：传输层（Transport Layer）
        提供端到端的数据传输服务，确保数据的可靠性和完整性。
        协议示例：TCP（Transmission Control Protocol）、UDP（User Datagram Protocol）、SCTP等。
